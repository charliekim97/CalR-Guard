# Public release checklist

Before making the repository public, confirm the following:

- [ ] repository owner name is correct
- [ ] professor / lab is comfortable with public release timing
- [ ] maintainer contact method is defined
- [ ] license choice is deliberate, not accidental
- [ ] `CITATION.cff` has real author names and version metadata
- [ ] README examples do not contain sensitive lab identifiers
- [ ] example files are safe to publish
- [ ] repository description and topics are set on GitHub
- [ ] security features are enabled on GitHub
