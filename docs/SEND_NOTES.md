# What to send to a professor

## Send these files

1. `calr_guard_release_v3.zip`
2. `docs/PROFESSOR_BRIEF.md`
3. `calr_guard_example_output_bundle_v3.zip`

## What to say in one paragraph

I built this as a companion layer before CalR rather than a CalR rewrite. The goal is to reduce import failures, standardize metadata entry, support variable/time-window exclusions, generate suggested exclusion windows from QC spikes, and leave a clear audit trail. The package includes a local app, a CLI, config-driven overrides, metadata-sheet merge support, and an export bundle with a human-readable review report.

## What not to say

Do not say:
- this replaces CalR
- this is production-ready
- this solves statistics better than CalR
- this is finished

## Better framing

Say:
- working beta
- companion preflight / curation tool
- aimed at reducing support burden and improving reproducibility
- designed for testing on real lab files next
