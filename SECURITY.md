# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Instead, contact the repository maintainer privately and include:
- a short summary of the issue
- affected file or component
- reproduction steps if available
- whether any credentials or sensitive data may be exposed

## Scope

This project is a local data-processing companion tool. The most realistic security risks are:
- accidental inclusion of lab data in issues or pull requests
- accidental commit of tokens, secrets, or credentials
- unsafe handling of uploaded files in future hosted deployments

## Current recommendation

Use a private repository until maintainer identity, release policy, and hosting posture are finalized.
