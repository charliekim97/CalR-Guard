# Code of Conduct

## Our standard

Treat collaborators with respect. Critique ideas, code, and workflow assumptions directly, but do not make it personal.

Expected behavior:
- be specific about bugs and failure cases
- share reproducible examples when possible
- assume mistakes are fixable
- keep discussion focused on the tool and the data workflow

Unacceptable behavior:
- harassment, personal attacks, or discriminatory language
- posting confidential or identifiable lab data without permission
- derailing technical discussion with bad-faith arguments

## Scope

This applies to repository discussions, issues, pull requests, and any project-linked communication.

## Enforcement

Repository maintainers may remove content or restrict participation when behavior makes collaboration worse instead of better.
